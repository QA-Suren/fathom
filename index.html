Fathom

Fathom is an opinionated, modular, & injectable foundation for building microservices on a JVM.

Microservice Foundation

Fathom provides a base to quickly bootstrap your microservice project using Undertow, Guice, Guava, Config, Args4j, Logback, and Commons-Daemon.

Fathom applications are developed-on and deployed-with Undertow. You do not build WARs with Fathom; you build integrated server applications.

Opinionated

  • Undertow is the development & deployment engine
  • Guice is the dependency injection mechanism
  • Guava is the standard library
  • Config is the configuration file parser
  • Args4j is the command-line parsing framework
  • Logback is the logging framework
  • Commons-Daemon is the service control integration
  • Java 8 is the baseline JVM

Modular

Fathom features a formal Module spec, Service classes, and dependency injection combined with the Service Locator design that gives you ultimate modularity.

Modules & services may be optionally disabled based on runtime mode and/or config settings.

There are several modules available to build your Fathom foundation:

  • JCache module for Hazelcast, Infinispan, or Ehcache.
  • Metrics module for runtime metrics collection
  • EventBus module for decoupled event passing
  • Mailer module for email sending
  • Quartz module for Quartz scheduling integration
  • REST module for RESTful routing & controllers based on Pippo
    • Security module for RESTful Authentication & Authorization against PAM, Windows, LDAP, JDBC, Redis, etc.
    • Shiro module for RESTful Authenication & Authorization
    • Multiple content-type engine choices
      • JAXB (default)
      • XStream
      • GSON
      • FastJSON
      • Jackson
      • SnakeYAML
    • Multiple template engine choices with Webjars support, localization (i18n), & pretty-time integration.
      • Freemarker
      • Jade
      • Pebble
      • Trimou
      • Groovy

Testing

  • Integration testing based on Rest-Assured.

Deployment

  • Five runtime modes: PROD, TEST, DEV, MASTER & SLAVE
  • Flexible config file with runtime-mode setting variations, property substitutions, & environment variable substitutions
  • Integrated support for HTTPS, HTTP, AJP, & HTTP/2 transports
  • Integrated Commons-Daemon service support for Windows & UNIX
  • Distribution bundling with Stork

Footprint

The footprint of a Hello World Fathom servlet microservice is about 7MB.
